# AI Slop Detector

> A end-to-end machine learning pipeline to detect AI-generated text — from data collection to a fine-tuned BERT classifier.

---

## 📖 Overview

AI Slop Detector is a binary text classifier that distinguishes between **human-written** and **AI-generated** text. It is built as a LoRA adapter on top of `bert-base-cased` and achieves **96% validation accuracy** on a dataset of Wikipedia paragraphs and their AI-generated rewrites.

## Data

### Human-Written Text

Human text was sourced by crawling and scraping Wikipedia.

- Two random paragraphs were stored per page visited
- **10,001 paragraphs** collected in total
- Scraping code: [`/scraping`](./scraping)
- Raw output: [`/scrapes`](./scrapes)

### AI-Generated Text

Each Wikipedia paragraph was rewritten by an LLM using a **two-pass generation strategy** designed to reflect real-world AI writing:

1. **Pass 1 — Summarize**: Condense the paragraph
2. **Pass 2 — Rewrite**: Expand into a new paragraph in a separate context

**Model used:** `GPT-5 Nano` — cost-effective, capable, and the most widely deployed model family in practice.  
**API:** OpenAI Batch API for efficient large-scale generation.  
**Code:** [`/data-generation`](./data-generation)

### Full Dataset

The final paired dataset (human + AI paragraphs) is publicly available:

- 🤗 HuggingFace: [`gouwsxander/wikipedia-human-ai`](https://huggingface.co/datasets/gouwsxander/wikipedia-human-ai)
- Local: [`/data/wikipedia.jsonl`](./data/wikipedia.jsonl)

##  Model

The classifier is a **LoRA adapter** of [`bert-base-cased`](https://huggingface.co/bert-base-cased) with a binary classification head, trained using the HuggingFace `transformers` and `peft` libraries.

| Metric              | Value |
|---------------------|-------|
| Validation Accuracy | **96%** |
| Base Model          | `bert-base-cased` |
| Adapter Type        | LoRA  |
| Context Window      | 512 tokens |

Training code: [`/training`](./training)

### 🚀 Quick Start

```python
from transformers import pipeline

classifier = pipeline(
    "text-classification",
    model="gouwsxander/slop-detector-bert",
)

texts = [
    "The Eiffel Tower was constructed between 1887 and 1889...",
    "The Eiffel Tower, an iconic symbol of French ingenuity, stands as a testament to..."
]

results = classifier(texts)
# LABEL_0 → Human-written
# LABEL_1 → AI-generated
```

> **Note:** You may see a warning about uninitialized weights during loading — this is expected and can be safely ignored.

**Label Reference:**
- `LABEL_0` — Human-written text
- `LABEL_1` — AI-generated text

## 📊 Benchmark Performance

The model was evaluated on the [RAID benchmark](https://github.com/liamdugan/raid) training set to assess generalization across writing styles and model families.

### TPR @ FPR = 1%

| Domain              | TPR @ FPR=1% |
|---------------------|--------------|
| IMDB Movie Reviews  | ~90%         |
| Wikipedia           | 30–40%       |
| arXiv Abstracts     | 30–40%       |
| News Articles       | 30–40%       |
| Book Excerpts       | 1–20%        |
| Recipes             | 1–20%        |
| Poetry              | 1–20%        |
| Reddit Posts        | 1–20%        |

The model outperformed random guessing across **all model families** and **most domains**, confirming that general, transferable features of AI writing were learned.

**Notable finding:** Text generated by **LLaMA** was the most reliably detected.

Evaluation code and full results: [`/raid`](./raid)

## ⚠️ Limitations

This model has several known limitations to keep in mind before deployment:

- **Narrow training domain** — The dataset consists exclusively of Wikipedia-style text; performance on other genres may vary.
- **Single AI source** — All AI-generated training examples come from GPT-5 Nano; the model may generalize less well to other generators.
- **Wikipedia purity assumption** — We assume Wikipedia is entirely human-written, which may not hold perfectly.
- **Short context window** — The model handles a maximum of **512 tokens**; longer texts must be chunked.
- **Paragraph-level training** — The model was trained on single paragraphs, not full documents.
